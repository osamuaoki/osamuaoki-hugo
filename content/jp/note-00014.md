AVRチップのISP経由プログラムの練習をします。これには、
[TTL-232R-5V を使った自作プロクラマー](https://github.com/osamuaoki/avrdude-friend)
を使うのも良いのですが、最近Atmel AVR ISP mkIIをただで人からもらったので
それを使います。

ATmega328P は手元にあったブレッドボードに挿せるArduino Nanoのコンパチボードや本物の
Uno等を使ってます。

## ソフト環境


```
$ sudo apt install avrdude
```

## 接続

AVRISP mkIIのVCCは電源電圧監視用でAVRISP mkII側からは電源供給されませんので、
別途チップへの電源供給ソースが必要！Unoなら12V、NanoならUSBを必ずつなぐこと。

不便なのでAVRISP mkIIのUSB 電源(TP1) と PIN#1 をつなぐ、５V電源供給ソースと
する等の改造記事もよくあります。

* https://qiita.com/hideyuki/items/71381b741d89878e3e79 Easy access via TP1
* http://shiozoku.blogspot.com/2014/02/avr-isp-mkii.html Use switch
* https://jeelabs.org/2010/04/02/avrisp-mkii-w-5v-power/ After poly fuse
* http://www.webx.dk/avrisp-mk2-modding/ 3V/5V selectable

ただし、ターゲットボード側で電源を確保している場合は使えません。

また、これはあとで考えましょう。

[TTL-232R-5V を使った自作プロクラマー](https://github.com/osamuaoki/avrdude-friend)
は電源電圧供給していた。

ISPケーブルの赤線は１ピン側です。

### Fuse

なぜかFuseが違います:

* Uno:  E:FD, H:DE, L:FF
  * SPIEN (Enable Serial programming and Data Downloading) On = 0
* Nano: E:FD, H:DA, L:FF
  * SPIEN (Enable Serial programming and Data Downloading) Off = 1

前に壊したかも知れないので、とりあえずNanoを修正:

```
$ sudo avrdude -c avrisp2 -P usb -p m328p -F  -U lfuse:w:0xfd:m -U hfuse:w:0xde:m -U efuse:w:0xff:m
```

なぜか書き換えられました。
